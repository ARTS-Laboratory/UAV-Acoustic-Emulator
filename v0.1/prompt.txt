# Instruction Set for an AI to Build a MATLAB UAV Acoustic Emulator

## Goal

Build a MATLAB toolbox that simulates microphone time-series signals generated by a UAV flying through 3D space above a 2D microphone plane.

The toolbox should let a user:

* enter any number of microphone locations in the XY plane,
* define a UAV trajectory in 3D,
* generate a time series at each microphone,
* save the signals for later algorithm development,
* include realistic but not overly complex physics such as attenuation, delay, noise, and optional Doppler shift.

This is a simulator, not the detection algorithm itself. The output should be physically plausible enough for downstream work such as TDA, classification, or distributed sensing experiments.

---

## Primary Requirements

### 1. Geometry

* Microphones are fixed on a 2D plane, assumed to be `z = 0`.
* Each microphone has an `(x, y)` location.
* The UAV has a time-varying `(x, y, z)` trajectory.
* The code must support any number of microphones, including small arrays such as 5 sensors or larger layouts such as 20 sensors.

### 2. Output

For every microphone, the toolbox must generate:

* a sampled time-series signal,
* the corresponding simulation time vector,
* metadata describing the UAV path, sensor geometry, and simulation settings.

The signals should be returned in a matrix, for example:

* `Y(:, i)` = signal at microphone `i`

### 3. Signal Model

At minimum, the received signal at microphone `i` should include:

* propagation delay,
* distance-based attenuation,
* additive noise.

Optional realism additions:

* Doppler shift from UAV motion,
* simple ground reflection,
* amplitude variation from altitude and range,
* sensor dropout or missing-node simulation.

### 4. Realism Level

Target “about 80% realism,” not full physical fidelity.

The simulator should be realistic enough that:

* signals change with range,
* channels differ from one another,
* motion affects timing and frequency,
* noisy conditions can obscure the source,
* the outputs still remain numerically stable and reproducible.

Do not overcomplicate the physics if it makes the toolbox harder to use.

---

## Core Mathematical Model

For each microphone `i`, generate:

[
y_i(t) = \frac{1}{r_i(t)^\alpha} s(t - \tau_i(t)) + n_i(t)
]

where:

* `r_i(t)` is the distance between the UAV and microphone `i`,
* `alpha` is the attenuation exponent,
* `tau_i(t) = r_i(t) / c`,
* `c` is speed of sound,
* `s(t)` is the source waveform,
* `n_i(t)` is noise.

If Doppler is enabled, modify the source frequency or time base using relative radial velocity.

---

## Inputs

The toolbox should accept the following inputs:

### Microphones

* `micXY`: `N x 2` array of microphone coordinates in meters.

### UAV Trajectory

* `uavXYZ`: `T x 3` array of UAV positions over time in meters.
* `t`: time vector in seconds.

### Acoustic Settings

* `fs`: sampling rate.
* `c`: speed of sound.
* `alpha`: attenuation exponent.
* `snrDb`: desired signal-to-noise ratio.
* `enableDoppler`: true/false.
* `enableReflection`: true/false.

### Source Settings

* Source waveform type:

  * tone,
  * multi-tone,
  * rotor-like harmonic signal,
  * broadband noise burst,
  * user-supplied waveform.

### Environment Settings

* background noise type,
* ambient noise level,
* optional wind-like low-frequency noise,
* optional sensor failure or dropout rate.

---

## Outputs

The toolbox should return:

* `Y`: `numSamples x numMics` matrix of simulated microphone signals,
* `t`: time vector,
* `meta`: structure containing:

  * microphone coordinates,
  * UAV trajectory,
  * source settings,
  * attenuation parameters,
  * noise settings,
  * simulation time,
  * any enabled effects.

Optional outputs:

* `cleanY`: microphone signals without noise,
* `distances`: UAV-to-mic distance over time,
* `delays`: propagation delay over time,
* `doppler`: Doppler shift estimate over time.

---

## Functional Modules

The AI should build the toolbox in modular form with the following functions:

### 1. `simulate_uav_acoustics.m`

Main entry point.

Responsibilities:

* validate input geometry,
* call helper functions,
* generate all microphone signals,
* package outputs.

### 2. `generate_uav_trajectory.m`

Create or accept a 3D UAV path.

Support:

* straight-line paths,
* arcs,
* loitering patterns,
* piecewise paths with turns and altitude changes.

### 3. `generate_source_signal.m`

Create a UAV acoustic source waveform.

Start simple with:

* one tone,
* multiple harmonics,
* tone plus broadband noise.

### 4. `propagate_to_microphones.m`

Compute:

* distance,
* delay,
* attenuation,
* optional Doppler,
* optional reflection.

### 5. `add_noise.m`

Add ambient noise to each microphone channel with a controllable SNR.

### 6. `save_simulation.m`

Save outputs, metadata, and settings to MAT files.

### 7. `plot_simulation_results.m`

Plot:

* UAV trajectory,
* microphone layout,
* sample waveforms,
* spectrograms if desired.

## Physics and Signal Rules

### 1. Distance

For microphone `i` at `(x_i, y_i, 0)` and UAV position `(x_u, y_u, z_u)`:

[
r_i(t)=\sqrt{(x_u-x_i)^2+(y_u-y_i)^2+(z_u)^2}
]

### 2. Delay

[
\tau_i(t)=\frac{r_i(t)}{c}
]

### 3. Attenuation

Use a user-selectable attenuation model, starting with:

* inverse distance, or
* inverse distance squared.

### 4. Doppler

If enabled, estimate the radial velocity and adjust the signal frequency accordingly.

### 5. Noise

Add random noise with configurable SNR and optional colored spectrum.

### 6. Reflection

If enabled, allow one simple ground reflection path as an optional second arrival.

---

## Software Design Rules

### The code should:

* be written in MATLAB,
* be easy to read and modify,
* avoid unnecessary toolboxes unless they are clearly helpful,
* work for arbitrary microphone counts,
* use vectorized operations where reasonable,
* include comments explaining assumptions,
* separate geometry, signal generation, and plotting.

### The code should not:

* assume only a fixed number of microphones,
* hard-code a single UAV path,
* require Simulink,
* depend on a centralized fusion node,
* implement detection or classification algorithms yet,
* try to maximize realism at the expense of clarity.

---

## Default Simulation Behavior

If the user provides minimal inputs, the toolbox should use sensible defaults:

* `c = 343 m/s`
* attenuation exponent `alpha = 1`
* no reflection by default
* Doppler off by default
* white Gaussian noise by default
* a simple straight-line UAV path by default

---

## Validation Checks

The AI should include basic validation logic:

* microphone coordinates must be numeric and finite,
* trajectory must have matching time samples,
* sampling rate must be positive,
* SNR must be a valid numeric scalar,
* all output arrays must be dimensionally consistent.

---

## Recommended Development Order

1. Build the geometry and signal output pipeline.
2. Add attenuation and propagation delay.
3. Add noise.
4. Add optional Doppler.
5. Add optional reflection.
6. Add plotting and save functions.
7. Test with 5 microphones and a simple UAV path.
8. Expand to 20 microphones and multiple UAV flight paths.

---

## Acceptance Criteria

The toolbox is complete when it can:

* accept arbitrary 2D microphone locations,
* simulate a UAV moving through 3D space,
* output one time series per microphone,
* generate visibly different signals at different sensors,
* run repeatably with a random seed,
* be used as a source of synthetic data for future algorithms.

---

## Suggested Name

A good working name for the toolbox is:

**MATLAB UAV Acoustic Emulator**

or

**Distributed UAV Acoustic Simulation Toolbox**

---

## One-Sentence Instruction to the AI Builder

Write a MATLAB-based, script-friendly UAV acoustic simulator that takes microphone XY coordinates and a 3D UAV trajectory as input and returns one noisy time-series signal per microphone using distance-based attenuation, propagation delay, and optional Doppler or reflection effects.
